Part 1: Identifying Usability Problems

Picking a Website

I chose to redesign the home page of Table570, one of the local restaurant from my high school:

Finding Problems

The website had some interface flaws that could confuse the users:
  1. In the navigation bar, there are buttons with similar or overlapping functionalities, such as delivery and order online.
  2. Order pickup and order delivery buttons below the navigation bar also does not seem necessary given the similar options in the navigation bar.
  3. Hours of operation part is unclear; so is the hours on the website temporary or set?
  4. Why is hours of operation separate from contact and location part? Don't people look for them all at the once?
  5. In the same part, dash is difficult to see. It almost touches the 'M'
  6. There is a space missing between ‘Sun-Thurs’ and ‘|’ in the hours or operation part.
  7. In the footer, contact and location part is not aligned correctly.
  8. In the contact part, dots are difficult to see in the phone numbers.
  9. Instagram link only shows up in mobile devices but it also shows email and phone call shortcut icons along with the link, which seems unnecessary given the contact information already in the website.

Accessibility

The screen reader reads the website in F-shaped visual flow; first reads the navigation bar left to right, then reads order pick up and order delivery, and so on. The website does not have any accessibility concerns. However, it has 16 contrast errors due to low contrast between text and background colors and 7 alerts due to pdf links, underlined or alternative texts, missing heading, email plugins, and links to ordering/delivery websites.

Annotations:
  1. Since width of mobiles is significantly smaller than that of pads and desktops, I made a menu button(not functional) holding information in the navigation bar instead of original navigation bar.
  2. For the main image below the navigation bar, I used CSS flexbox, width, and height functionalities to help it fit to difference device sizes.
  3. For the hiring tab, I used CSS flexbox, width, and text-align functionalities to set the text to center.
  4. I moved hours part to the footer because these information are similar; people look for these information don't look for these information separately but at the same time.

Annotations:
  1. Social media buttons only showed up when opened from mobile. So in order to prevent this, I added it to the contact information because social media is a part of contact information.
  2. In order to prevent not using empty space, I used flexbox to keep track of the 4 information in the footer. Since the pad width is greater than mobile width, there is now more width space for locations and hours parts.
  3. Since width is greater than mobile, full navigation bar is used instead of menu button.

Annotations:
  1. Similarly from pad, using flexboxes helps move hours part if there is more width space.
  2. If the website is fully functional, then clicking navigation bar, buttons, social media logos, application text, and email will work as it should be.
